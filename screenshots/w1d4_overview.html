// 终态自动 close（不再重连）
this.ws.onmessage = (ev) => {
  const msg = JSON.parse(ev.data)
  emit(msg)
  if (msg.type === 'SUCCESS' ||
      msg.type === 'FAILURE' ||
      msg.type === 'REVOKED') {
    this.receivedFinal = true
    this.close()
  }
}

// handleClose: 指数退避重连
private handleClose() {
  if (closedByUser || receivedFinal) return
  if (attempt >= maxReconnect) { state = 'failed'; return }
  const backoff = Math.min(Math.pow(2, attempt) * 1000, maxBackoffMs)
  setTimeout(() => this.connect(), backoff)
}

5backend/api/v1/ws.py · 2 个 WS 终态 Bug 修复

🐛 Bug A：终态 SUCCESS 发两遍（先空后带 result）
旧代码：subscribe_progress 收 SUCCESS → 立即 send_json(msg) → reload task → 改 msg → 再 send 一次
E2E 第一遍 SUCCESS 收到就 break 断开，第二遍 retry 完才发——客户端看不到
✅ 改：SUCCESS 不立即发 → retry 10×200ms 拿 result_json → 一次性 send 带 result 的 SUCCESS → break
🐛 Bug B：SUCCESS 时 reload task 拿 result_json 失败（commit 竞态）
WS 收 SUCCESS 信号时，DB 事务可能还没 commit result_json（~10-50ms 窗口）
30% 概率 SUCCESS 不带 result，前端拿不到曲线
✅ 改：retry for i in range(10): await asyncio.sleep(0.2) 兜底 2s，100% 带 result
