## 4. 关键技术决策（W1 周级）

### 4.1 决策 #1：CDN 模式 vs npm 模式
- **决策**：**保留 CDN 模式**（element-plus / icons-vue / axios / vue-i18n 走 cdn.jsdelivr.net）
- **理由**：element-plus 在 sandbox FUSE 上有 **122 个 0 字节 .mjs 文件**（npm install 大包截断老问题），切回 npm 触发 esbuild 77 errors
- **代价**：依赖外网（cdn.jsdelivr.net）；本机无网时 vite 启动失败
- **下次切回时机**：M4 Docker 化时，在镜像里 `npm install` 不受 sandbox FUSE 影响

### 4.2 决策 #2：WS 断线重连策略
- **决策**：**指数退避 1s→30s × 8 = 121s 总窗口**，终态自动 close 不重连
- **理由**：覆盖 sandbox 4h 回收 + 短时网络抖动；不无限重试（避免资源耗尽）
- **简化**：**后端不加心跳**——依赖浏览器 WebSocket 协议层 ping/pong + 客户端重连兜底

### 4.3 决策 #3：WS SUCCESS 一次性发带 result
- **决策**：后端 SUCCESS **不立即发**，retry 10×200ms 拿 result_json → **一次性发带 result**
- **理由**：解决 2 个 bug：
  - Bug A：原代码 SUCCESS 发两遍（先空后带 result），客户端第一遍就 break
  - Bug B：SUCCESS 时 commit 竞态（30% 概率 result_json 还没 commit）
- **收益**：客户端 100% 收到完整 result，E2E 简化

### 4.4 决策 #4：i18n 双语 13 模块
- **决策**：zh-CN / en-US **154 嵌套 key 完全对齐**，13 顶层模块
- **理由**：13 模块（app / home / common / error / designer / quality / optimize / optimizeDetail / storyboard / storyboardDetail / projects / projectDetail / errors）—— 每个 Vue 页面都有对应 key 集合
- **修复**：D5 修了 1 个历史 bug（D2 时代遗留的孤立 `} as const` 切断业务页面 key）

### 4.5 决策 #5：sandbox 4h 回收恢复脚本
- **决策**：4 服务全启脚本沉淀为标准流程（uvicorn / Celery / Redis / Vite）
- **工具**：使用 `setsid nohup` + `</dev/null` 位置参数传 cwd（避免 sandbox 重置 cwd）
- **依赖安装**：pip 装 `celery redis trimesh numpy scipy loguru websockets pydantic-settings` 全套
