### 前端部署修复（v6 → v7）

**问题**：W1-D5 部署后前端白屏，Console 报 `Cannot destructure property 'Component' of 'undefined'`

**根因**：`inline-vendor.js` 只包含 Vue/axios/VueI18n/Icons，缺少 VueRouter、Pinia、ElementPlus JS，导致运行时 `window.VueRouter` 为 undefined → router 创建失败

**修复路径**（共 5 轮迭代）：
1. 修法 1-3：尝试补全 inline-vendor.js 的 vendor 列表 → 双 Vue 实例冲突，`currentInstance` 变量不互通
2. 修法 4：11 个 .vue 文件改 `window.*` → stores 没改 + VueRouter/Pinia 不在 vendor → 仍失败
3. **修法 5（v6 最终方案）**：彻底放弃 window 全局对象方案，回归标准 npm import + Vite 打包
   - 重写 `main.ts`、`App.vue`、`router/index.ts`、`i18n/index.ts`
   - 删除旧 `vite.config.js`（含 shim alias）+ 所有 shim 文件
   - `vite.config.ts` 纯净版：仅 `@` alias + manualChunks 分包（vendor-vue / vendor-ui / vendor-3d）
   - Build 成功：1545 modules transformed，4 个 vendor chunk + 11 个 lazy page chunks

**v7 迭代**：左侧参数面板布局优化
- 5 个独立卡片合并为 1 个可折叠参数面板
- 5 个分组（基础尺寸/比例姿态/曲面特征/玻璃与轮/灯）支持点击折叠
- 默认展开「基础尺寸」和「比例姿态」，其余收起
- 支持全部展开/全部收起快捷按钮
