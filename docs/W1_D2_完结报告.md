## 4. 关键技术决策

### 4.1 vue-i18n 走 CDN
跟 W1-D1 同样教训：**npm install 大包 → FUSE 0 字节截断**。vue-i18n@9.14.5 npm 安装 OK（0 个 0 字节），但保险起见走 CDN 引入（jsdelivr `vue-i18n.global.prod.js`）。

**npm 模式 vs CDN 模式对比**：
- npm 模式：可以 import 享受 TypeScript 类型提示
- CDN 模式：依赖 window 全局对象，无类型提示，但避免 FUSE 0 字节风险
- **W1 完工后切回 npm**：4 行 diff（删 CDN script + main.ts 改回 import + index.html 删 script 标签）

### 4.2 i18n 自动检测 + 手动切换
```ts
// 优先级：localStorage > navigator.language > zh-CN
function resolveLocale() {
  const stored = localStorage.getItem('evolution_ai_lang')
  if (stored === 'zh-CN' || stored === 'en-US') return stored
  if (navigator.language.toLowerCase().startsWith('en')) return 'en-US'
  return 'zh-CN'
}
```

主人在新浏览器首次访问 → 自动检测浏览器语言。手动切换后 → localStorage 记忆，**主人重打开浏览器也保持**。

### 4.3 axios 拦截器
**请求拦截**：
1. 注入 token（localStorage.getItem('evolution_ai_token')）
2. 启动 loading（按 URL 去重，Pinia store 联动）

**响应拦截**：
1. 成功 → 关 loading
2. 失败 → 归一化 8 case 错误 → 弹 ElMessage → 关 loading

**NormalizeError 8 case**：
1. 网络错误（无 response）→ "网络错误，请检查连接"
2. 超时（ECONNABORTED）→ "请求超时，请稍后重试"
3. 4xx 客户端错误（401/404/422）→ "请求错误 (status)"
4. 5xx 服务端错误 → "服务器错误 (status)"
5. 业务错误（后端 200 + error_code 非 0）→ 透传 message
6. WebSocket 握手错误
7. JSON 解析错误 → "服务器返回数据格式错误"
8. 未知错误 → "未知错误"

### 4.4 Pinia loading store（按 URL 去重）
```ts
const inflight = ref<Record<string, number>>({})
function start(key) { inflight.value[key] = (inflight.value[key] ?? 0) + 1 }
function stop(key) { if (--inflight.value[key] <= 0) delete inflight.value[key] }
const isLoading = computed(() => Object.values(inflight.value).some(n => n > 0))
```

**设计要点**：同一 URL 的并发请求只触发 1 次 loading（计数器 +1/-1，归 0 时删除 key），不会因为 N 个并发请求导致 loading 闪烁 N 次。

### 4.5 vite proxy 加 /health
**W1-D1 遗留 bug**：vite proxy 只代理 `/api/*`，但 Home.vue 调 `/api/v1/health` 是**错的**——后端 `/health` 不在 `/api/v1` 前缀下。

W1-D2 修：
- Home.vue: `axios.get('/api/v1/health')` → `axios.get('/health')`（绝对路径跳过 baseURL）
- vite.config.ts: 加 `/health` 代理 → 浏览器里调 `/health` 也能 vite proxy 到 8000
