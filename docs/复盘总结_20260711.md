### 3.2 失败分析

**第13页（迭代式控制点坐标优化）**：
- 6次尝试均触发 API 5000 错误
- 尝试了多种简化策略（减少内容、调整布局、更换标题），均触发同样的5000错误
- **结论**：生图API服务端对特定内容的渲染存在偶发故障，与 prompt 复杂度无直接关系

**第22页（第一阶段 Q3-Q4）**：
- 第1批失败，第2批简化后成功
- 原内容包含较多时间节点和任务列表，简化后通过

### 3.3 经验沉淀

| 经验 | 说明 |
|------|------|
| 失败页重试策略 | 首次失败后简化 prompt 重试，3轮仍失败则标记为服务端故障 |

## 四、Streamlit 渲染问题复盘

### 4.1 问题定位过程

| 阶段 | 操作 | 结论 |
|------|------|------|
| 用户反馈 | "没有封闭汽车形态" | 初步怀疑代码逻辑 |
| 代码审查 | 5轮 grep + read_file | 代码逻辑正确 |
| 环境检查 | Plotly 6.8.0 vs Streamlit 1.57.0 | **根因：版本不兼容** |
| 修复方案 | 降级 Plotly → 5.24.1 | 用户本地降级成功 |
| 启动问题 | `streamlit` 不在 PATH | 改用 `python -m streamlit run app.py` |

### 4.2 根因

Plotly 6.8.0 (Plotly.js 3.6.0) 与 Streamlit 1.57.0 (旧版 Plotly.js) 的 schema 不兼容，导致渲染白屏。降级 Plotly 到 5.24.1 后兼容。

### 4.3 教训

- **依赖版本必须锁定**：`requirements.txt` 已更新为 `plotly>=5.18,<6.0`
- **环境差异是白屏首因**：沙箱 Plotly 6.8.0 正常，用户本地 Streamlit 版本不匹配
- **代码审查不能替代环境验证**：代码逻辑正确 ≠ 运行正确
