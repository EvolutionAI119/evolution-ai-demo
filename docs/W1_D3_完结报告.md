## 4. 5 项技术决策

### 4.1 路由分 11 条（继承 W1-D1 6 路由 + 4 详情子页 + 2 错误页 + 1 兜底 redirect）

为什么这么分：
- 5 静态路由：home / designer / quality / optimize / storyboard / projects
- 4 :id 详情：OptimizeDetail(:taskId) / StoryboardDetail(:projectId) / ProjectDetail(:projectId) / 暂未启用(:id quality)
- 2 错误页：404 / 500
- 1 兜底：`/:pathMatch(.*)*` → /404（任意未匹配路径重定向到 404）

### 4.2 Designer 用 11 维 CarParams 简化版

完整 CarParams 22 维给后端 /api/v1/car/build 入参，前端简化为视觉/调整用 11 维：
- L/W/H/wheelbase/前后悬/离地间隙/车顶（7 个尺寸参数）
- 3 个角度（front_overhang_angle / rear_overhang_angle / roof_angle）

**原因**：22 维滑块体验差，11 维覆盖主要调整 + 后端默认值兜底。

### 4.3 OptimizeDetail 用轮询，WS 留 W1-D4

```ts
// OptimizeDetail.vue 每 1s 调 /api/v1/task/{task_id}
setInterval(async () => {
  const r = await axios.get(`/api/v1/task/${taskId}`)
  taskInfo.value = r.data
  if (['SUCCESS', 'FAILURE'].includes(r.data.status)) {
    clearInterval(poll)
  }
}, 1000)
```

**为什么本轮不接 WS**：W1-D4 才在 `ws.ts` 接入，前端 OptimizeDetail.vue 改造只需替换轮询为 `onProgress` 回调。

### 4.4 Optimize 页调 /api/v1/optimize/start-preset 后跳 /optimize/{task_id}

**为什么跳详情页**：主人看 demo 时更直观——提交后立刻看 PENDING→STARTED→SUCCESS 时间线，胜过历史列表里翻。

### 4.5 错误页 500 5s 自动 redirect 回首页

```ts
// ServerError.vue
setTimeout(() => router.push('/'), 5000)
```

**为什么 5s**：sandbox 4h 回收 + 主人刷新时可能撞到 500，5s 后给主人重新进比停死页强。

## 5. 关键 Bug 修复

### E2E 第二轮 7 失败 → 0 失败

**Bug A**：`/api/v1/openapi.json` 错路径

后端 openapi.json **不在** /api/v1 前缀下，正确路径是 `/openapi.json`（与 W1-D2 Home.vue 路径 bug 同源）。

**修复**：
```python
- code, body = http_get(f"{API}/api/v1/openapi.json")
+ code, body = http_get(f"{API}/openapi.json")
```

**Bug B**：`expected_paths` 不带 `/api/v1` 前缀

openapi.json 里 paths 是带 `/api/v1` 前缀的完整路径，不是裸路径。

**修复**：
```python
- expected_paths = ["/car/build", "/quality/assess-preset", ...]
+ expected_paths = ["/api/v1/car/build", "/api/v1/quality/assess-preset", ...]
```

**Bug C**：`/api/v1/project/` 含尾斜杠

实际路径是 `/api/v1/project/{project_id}`，无尾斜杠变体。

**修复**：
```python
- "/api/v1/project/",
+ "/api/v1/project/{project_id}",
```
